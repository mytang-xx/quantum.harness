The following parameters are needed to specify a variational calculation. 1. length of a variational trajectory: this is the total number of optimization steps used to reach convergence in a variational calculation. 2. MC sweeps/vstep: this is the number of MC sweeps needed to compute the ensemble average in Eqs. 9-10 for one optimization step. Note that this number is typically quite small. It is a feature of the method that the gradient and the Hessian do not need to be computed with great statistical accuracy. The noisiness of the gradient and of the Hessian are averaged out during a variational trajectory. This is the key tenet in stochastic optimization: even with an inaccurate gradient at each step, the result can still be statistically accurate in the long run. 3. µ : this is the step size of the gradient descent. 4. nw : this is the number of multiple walkers in the sampling of gradient and Hessian in each optimization step. **VII.** **TEMPORAL CORRELATION OF MONTE CARLO CONFIGURATIONS IN**

**THE UNBIASED AND BIASED SIMULATION**
### Here we measure the correlation time of an estimator in an MC simulation with the block averaging method [S3]. In this method, a block-averaged trajectory is obtained by first partitioning the original MC trajectory into non-overlapping blocks of the same size and taking the averages of these blocks as the new blocked data points. Standard errors are then computed for the block-averaged trajectories for different block sizes. The speed with which the standard errors converge with respect to block size is a measure of the temporal correlation of the MC trajectory. In fact, [S3] shows that the standard errors converge exponentially to the true statistical uncertainty of the estimator as the block size increases and the time scale of this exponential function is the correlation time of the original MC trajectory.
