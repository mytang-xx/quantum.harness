# Pauli-Markov Sampling

Markov-chain Metropolis sampling on Pauli strings, used to estimate Pauli-weighted observables (notably stabilizer Rényi entropies and long-range magic) from any wavefunction representation that exposes Pauli expectation values `Tr(ρ P)` (DMRG / MPS, TTN, ED, or experimental measurements).

This card is the algorithm card per the method-card-per-algorithm rule. It bundles three runtime variants that share the chain machinery:

- **Pauli-Markov via tensor network** — exact `Tr(ρ P)` from the tensor-network wavefunction (DMRG/MPS or TTN); stochastic sampling at the level of the Pauli operator.
- **Deterministic Pauli-basis lift** — represent the MPS in the Pauli basis directly and contract; no Markov chain. Trades sampling cost for an additional MPS contraction; included here as a runtime variant rather than a separate card because it shares the convention card and benchmark targets.
- **Experimental-protocol variant** — `Tr(ρ P)` is replaced by a finite-shot estimate `P̄` from `N_M` measurements per Pauli; same chain machinery on top.

## Setup

- For the tensor-network and deterministic-lift variants: `make install julia && make install itensors`.
- TTN-based proposals require an additional TTN package; see `knowledge-base/methods/ttn.md`.
- For the experimental-protocol variant the data are measurement records; the harness only runs the chain on `P̄`.

## Notation

- `ρ`: density matrix of the target state (pure for ground-state work).
- `P ∈ P_N`: Pauli string of length `N` over the Pauli group of `N` qudits (`d=2` qubits, `d>2` clock-shift qudits — see `knowledge-base/magic-conventions.md`).
- `Π_P`: target distribution. Choices:
  - `Ξ_P = |Tr(ρP)|² / d^N` for full-state SRE (`M_n`).
  - `Π_{P,n} ∝ |Tr(ρP)|^{2n}` for the alternative (used in the increment/subleading construction).
  - Joint distribution on `P_AB = P_A ⊗ P_B` for long-range magic `L(ρ_AB)`.
- `N_S`: number of Pauli-string samples in the Markov chain.
- `N_M`: number of physical-measurement shots per Pauli (experimental-protocol variant only).
- `τ_int`: integrated autocorrelation time of the chain.
- `χ`: bond dimension of the underlying tensor network.

See `knowledge-base/magic-conventions.md` for the full convention list (qudit Pauli definitions, partition modes, sign normalizations).

## Estimator forms

Estimator definitions are physical content cited from `knowledge-base/magic-conventions.md`. This card declares which estimator each runtime variant evaluates and how the chain is driven.

| Quantity | Sampling distribution | Estimator role |
|---|---|---|
| `M_n`, `n ≠ 1` | `Ξ_P` | Reweighted SRE estimator. |
| `M_1` | `Ξ_P` | Logarithmic SRE estimator. |
| `M_n`, `n < 1` | `Π_{P,n}` | Reverse reweighting (better than `Ξ_P`). |
| `L(ρ_AB)` | `Ξ_{P_AB}` or `Π_{P_AB,2}` | Decompose `L = I_2 − W`; Rényi-2 mutual information `I_2` is a byproduct. |
| Subleading term `c_N` (1D) | `Π_{P,n}` | Joint sample on size-`N` and size-`N/2` partitions. |
| Higher-D subleading combinations | `Π_{P,n}` | Subsystem-difference linear combinations (the topological-entanglement-entropy family) cancel volume + area-law terms. |

## Proposal moves

The chain explores `P_N` (or the symmetry-restricted subgroup) by Metropolis updates:

- **Single-site update**: replace `P_i → P_i'` at one site.
- **Two-site update**: replace `P_i, P_j → P_i', P_j'`. Required when the state is invariant under a Z_n symmetry generated by `Π_i Z_i` — symmetry-preserving Pauli strings are generated by `Z_i` and `X_i^† X_j`. Two-site updates ensure ergodic exploration of the symmetric sector.
- **Time-reversal sector** (when applicable): for `d=2` real-amplitude states, restrict to even number of `Y`'s.

Acceptance probability is the standard Metropolis ratio `min(1, Π_{P'}/Π_P)`.

## Stages (multi-stage orchestration)

Per AGENTS.md "multi-stage orchestration lives in method cards", stages are declared explicitly. The generic `/run-stage` primitive executes a single stage; `/slurm-grid` parallelizes the parameter axis.

### Stage 0 — wavefunction prep (input to subsequent stages)

| Input | Output |
|---|---|
| Hamiltonian, sizes, parameters | Tensor-network wavefunction file (`results/<run>/state.h5` or equivalent), bond-dimension log. |

Driver: `knowledge-base/methods/dmrg.md` (MPS) or `knowledge-base/methods/ttn.md` (TTN). For the experimental-protocol variant, this stage is a measurement run on hardware.

### Stage 1 — single-size Pauli-Markov sampling

| Input | Output |
|---|---|
| Wavefunction file, partition spec, target distribution `Π`, `N_S`, `N_warmup`, proposal class | `results/<run>/N=<N>/chain.h5` with samples, log-weights, autocorrelation diagnostics. |

The driver runs Algorithm-1 Metropolis: compute `Tr(ρ P)`, propose `P'` from the proposal class, accept with the Metropolis ratio, accumulate the estimator. Report `τ_int` per estimator.

### Stage 2 — increment / subleading combination

| Input | Output |
|---|---|
| Stage-1 outputs at sizes `{N, N/2, N/4, …}` (1D) or `{L×L, L/2×L, L/2×L/2, …}` (2D) | Combined estimator value with propagated error bars. |

In 1D the combination is `c_N = 2 M_n(N/2) − M_n(N)` and `M_n(N) = Σ_k 2^k c_{N/2^k} + M_n(small)`. In 2D, use the subsystem-difference combinations from the topological-entanglement-entropy family: `γ M_n = M_n(L×L) − 4 M_n(L/2×L) + 4 M_n(L/2×L/2)` and similar. The combination cancels volume-law (1D) or volume + area-law (2D) terms so the residual is `O(1)` or `O(log N)` and the variance becomes polynomial in `N` even for `n > 1`.

### Stage 3 — convergence + variance diagnostics

| Input | Output |
|---|---|
| All chain outputs | `(N_S → N_S/2)` reblocked errors, `τ_int(N)` curve, `σ(N)` curve. |

Variance bounds are content; this card just declares the diagnostic. See "Variance bounds" below.

## Knobs

| Knob | Effect | Starting point |
|---|---|---|
| `N_S` | Total Markov-chain length. Statistical error `~ 1/√(N_S/τ_int)`. | `10⁵`–`10⁷`; set per the variance bound below and reblock to verify. |
| `N_warmup` | Burn-in. Discarded before estimator accumulation. | `~10 × τ_int`; conservative default `10⁴` then check. |
| Proposal class (single-site / two-site) | Single-site is fastest per step; two-site is required to preserve Z_n symmetry. | Two-site for any model with Z_n symmetry generated by `Π_i Z_i`; single-site otherwise. |
| `Π_P` (target distribution) | `Ξ_P` for `M_n` direct; `Π_{P,n}` for the increment construction. | Use `Π_{P,n}` whenever `n ≠ 1` and the target is `M_n` directly; switch to the increment construction when `M_n` is volume-law. |
| Reblocking block size | Statistical error estimate after correlated chain. | Block size `~10 × τ_int` is standard; sweep to verify the error has converged. |
| `N_M` (experimental variant only) | Finite-shot bias on `P̄ ≈ Tr(ρP)`. | Set so the bias `(1−⟨P⟩²)/N_M` is below the target accuracy on the SRE density. |

## Variance bounds (content per `knowledge-base/magic-conventions.md`)

- For `M_1`: `Var(M_1)` is at most quadratic in `N`; the estimator is *efficient* (polynomial cost). For `m_1 = M_1/N`, the standard error scales as `1/√N`.
- For `M_n`, `n ≠ 1`: when the SRE is volume-law (`M_n ~ N`), the variance grows exponentially with `N`. Use the increment / subleading construction (Stage 2) to cancel the volume-law term and recover polynomial cost. When the SRE is at most `O(log N)` (frustrated regimes, certain Bell-magic decompositions), the direct estimator is already efficient.
- For `L(ρ_AB)`: efficient because `L` is bounded by mutual-information-like content; volume-law cancellation is automatic.

## Pitfalls

- **Critical slowing-down at criticality** — `τ_int` grows near continuous QPTs. Diagnose via `σ(N)` curves; mitigate with cluster updates when implemented, or with longer chains and reblocked errors.
- **Sign of `L(ρ_AB)`** — SRE does not satisfy subadditivity, so `L(ρ_AB)` is not sign-definite. Negative extrema at criticality are valid; do not flip signs to "fix" them.
- **Wrong symmetry sector in proposals** — single-site proposals on a Z_n-symmetric state rapidly leave the symmetric subgroup and bias the estimator. Always use two-site proposals when symmetry-preserving Pauli strings require it.
- **`Π_{P,n}` with small support** — at large `N` and `n > 1`, `Π_{P,n}` concentrates on a tiny fraction of `P_N` and the chain may get stuck. The increment construction is the standard fix; cluster updates help further.
- **Finite-`N_M` bias** (experimental variant) — finite shots introduce `O(1/N_M)` bias that *does not* shrink with `N_S`. Both `N_M` and `N_S` must be balanced; treat them as two independent convergence parameters via `/verify-convergence`.

## Deterministic Pauli-basis lift (variant)

For MPS wavefunctions, an alternative to the Markov chain is to represent `Tr(ρP)` directly as an MPS over the Pauli index axis (`d^2`-dim per site) and contract with standard MPS routines. The resulting object — the *Pauli-basis MPS* — gives:

- **SRE** by treating `M_n` as a 2D tensor network and approximating its contraction with MPS algorithms.
- **Bell magic, stabilizer nullity** — additional monotones that the Markov-chain protocol does not directly access.
- **Stabilizer-group identification** — by reading off zero entries of the Pauli MPS.

The lift trades Markov-chain variance for tensor-network truncation error; convergence is governed by an additional bond dimension on the Pauli axis. It is the recommended runtime when (i) the MPS is already in hand, (ii) the target observable is a Pauli-MPS-friendly monotone (Bell magic, nullity), or (iii) volume-law magic with `n > 1` makes the Markov chain exponentially expensive even after the increment construction.

## Verification (per-method, complements skill-level verification)

- **Single-qudit limit** — for product states of single qudits, the chain estimator must reproduce the analytic value (see `knowledge-base/magic-conventions.md` qudit examples).
- **Stabilizer-state limit** — `M_n = 0` for any stabilizer state; chain estimator should return 0 within statistical error.
- **`χ`-convergence** — sweep the wavefunction's bond dimension and confirm the SRE estimate asymptotes; this is `/verify-convergence` over `χ` and is the diagnostic for FIG. 13-class checks.
- **`N_S`-convergence** — reblocked error decreases as `1/√(N_S)` once the chain is past `~10 × τ_int`.
- **Cross-method check** — TTN-based and MPS-based chains must agree on the same Hamiltonian within both methods' truncation budgets. The deterministic Pauli-basis lift is a third independent path.
