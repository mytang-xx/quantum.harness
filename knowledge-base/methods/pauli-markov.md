# Pauli-Markov Sampling

Markov-chain Metropolis sampling on Pauli strings, used to estimate Pauli-weighted observables (notably stabilizer R√©nyi entropies and long-range magic) from any wavefunction representation that exposes Pauli expectation values `Tr(œÅ P)` (DMRG / MPS, TTN, ED, or experimental measurements).

This card is the algorithm card per the method-card-per-algorithm rule. It bundles three runtime variants that share the chain machinery:

- **Pauli-Markov via tensor network** ‚Äî exact `Tr(œÅ P)` from the tensor-network wavefunction (DMRG/MPS or TTN); stochastic sampling at the level of the Pauli operator.
- **Deterministic Pauli-basis lift** ‚Äî represent the MPS in the Pauli basis directly and contract; no Markov chain. Trades sampling cost for an additional MPS contraction; included here as a runtime variant rather than a separate card because it shares the convention card and benchmark targets.
- **Experimental-protocol variant** ‚Äî `Tr(œÅ P)` is replaced by a finite-shot estimate `PÃÑ` from `N_M` measurements per Pauli; same chain machinery on top.

## Setup

- For the tensor-network and deterministic-lift variants: `make install julia && make install itensors`.
- TTN-based proposals require an additional TTN package; see `knowledge-base/methods/ttn.md`.
- For the experimental-protocol variant the data are measurement records; the harness only runs the chain on `PÃÑ`.

## Notation

- `œÅ`: density matrix of the target state (pure for ground-state work).
- `P ‚àà P_N`: Pauli string of length `N` over the Pauli group of `N` qudits (`d=2` qubits, `d>2` clock-shift qudits ‚Äî see `knowledge-base/magic-conventions.md`).
- `ŒÝ_P`: target distribution. Choices:
  - `Œû_P = |Tr(œÅP)|¬≤ / d^N` for full-state SRE (`M_n`).
  - `ŒÝ_{P,n} ‚àù |Tr(œÅP)|^{2n}` for the alternative (used in the increment/subleading construction).
  - Joint distribution on `P_AB = P_A ‚äó P_B` for long-range magic `L(œÅ_AB)`.
- `N_S`: number of Pauli-string samples in the Markov chain.
- `N_M`: number of physical-measurement shots per Pauli (experimental-protocol variant only).
- `œÑ_int`: integrated autocorrelation time of the chain.
- `œá`: bond dimension of the underlying tensor network.

See `knowledge-base/magic-conventions.md` for the full convention list (qudit Pauli definitions, partition modes, sign normalizations).

## Estimator forms

Estimator definitions are physical content cited from `knowledge-base/magic-conventions.md`. This card declares which estimator each runtime variant evaluates and how the chain is driven.

| Quantity | Sampling distribution | Estimator role |
|---|---|---|
| `M_n`, `n ‚âÝ 1` | `Œû_P` | Reweighted SRE estimator. |
| `M_1` | `Œû_P` | Logarithmic SRE estimator. |
| `M_n`, `n < 1` | `ŒÝ_{P,n}` | Reverse reweighting (better than `Œû_P`). |
| `L(œÅ_AB)` | `Œû_{P_AB}` or `ŒÝ_{P_AB,2}` | Decompose `L = I_2 ‚àí W`; R√©nyi-2 mutual information `I_2` is a byproduct. |
| Subleading term `c_N` (1D) | `ŒÝ_{P,n}` | Joint sample on size-`N` and size-`N/2` partitions. |
| Higher-D subleading combinations | `ŒÝ_{P,n}` | Subsystem-difference linear combinations (the topological-entanglement-entropy family) cancel volume + area-law terms. |

## Proposal moves

The chain explores `P_N` (or the symmetry-restricted subgroup) by Metropolis updates. For unrestricted targets, symmetric local replacements are acceptable. For symmetry-restricted targets, use generator multiplication, not arbitrary local replacement:

- **Single-site update**: replace `P_i ‚Üí P_i'` at one site.
- **Z_n symmetry update**: when the state is invariant under a Z_n symmetry generated by `ŒÝ_i Z_i`, generate candidates by multiplying the current string by `Z_i` or by `X_i^‚ÄÝ X_j`. For qubit Pauli codes `I=0, X=1, Y=2, Z=3`, this is `P_i ‚Üí P_i ‚äª 3` for the `Z_i` move and `P_i,P_j ‚Üí P_i ‚äª 1, P_j ‚äª 1` for the `X_i X_j` move, up to irrelevant phases. These moves are self-inverse and preserve detailed balance in the symmetric sector.
- **Time-reversal sector** (when applicable): for `d=2` real-amplitude states, explicitly reject candidates with an odd number of `Y`'s. Do not rely on numerically tiny Pauli expectations to reject them implicitly.

Implementations should treat these as caller-supplied filters, not estimator internals: the sampler receives a target-sector predicate for full-string proposals and a factor-sector predicate for any split-string numerator factors. A concrete workflow may pass the same predicate to both, pass different predicates for different subsystem states, or use the unrestricted predicate. The method card gives examples of valid filters; it is not a hardcoded type list.

Acceptance probability is the standard Metropolis ratio `min(1, ŒÝ_{P'}/ŒÝ_P)`.

## Stages (multi-stage orchestration)

Per AGENTS.md "multi-stage orchestration lives in method cards", stages are declared explicitly. The compute script reads its method-card-declared stage list, runs each stage, and writes a per-stage manifest (`results/<run>/cells/<cell_id>/<stage>.manifest.json`) ‚Äî plain shell, not a separate skill. For embarrassingly-parallel grids on a cluster, `/parameter-scan` composes with `/slurm` to submit + monitor + fetch.

### Stage 0 ‚Äî wavefunction prep (input to subsequent stages)

| Input | Output |
|---|---|
| Hamiltonian, sizes, parameters | Tensor-network wavefunction file (`results/<run>/state.h5` or equivalent), bond-dimension log. |

Driver: `knowledge-base/methods/dmrg.md` (MPS) or `knowledge-base/methods/ttn.md` (TTN). For the experimental-protocol variant, this stage is a measurement run on hardware.

For tensor-network wavefunctions, Pauli-Markov sampling is a **cached update** algorithm:

- **TTN backend**: cache coarse-grained link operators and update only the path from modified physical site(s) to the root.
- **MPS backend**: cache left/right Pauli environments for the current string; evaluate candidates by contracting only the interval between changed site(s), then refresh affected environment ranges after accepted updates.

Recomputing `Tr(œÅP)` from scratch after every local proposal is a diagnostic proxy, not the paper-grade methodology. Manifests must record the expectation backend (`ttn_cached_links`, `mps_cached_env`, exact ED fallback, etc.).

### Stage 1 ‚Äî single-size Pauli-Markov sampling

| Input | Output |
|---|---|
| Wavefunction file, partition spec, target distribution `ŒÝ`, `N_S`, `N_warmup`, proposal class | `results/<run>/N=<N>/chain.h5` with samples, log-weights, autocorrelation diagnostics. |

The driver runs Algorithm-1 Metropolis: compute `Tr(œÅ P)`, propose `P'` from the proposal class, accept with the Metropolis ratio, accumulate the estimator. Report `œÑ_int` per estimator.

### Stage 2 ‚Äî increment / subleading combination

| Input | Output |
|---|---|
| Stage-1 outputs at sizes `{N, N/2, N/4, ‚Ä¶}` (1D) or `{L√óL, L/2√óL, L/2√óL/2, ‚Ä¶}` (2D) | Combined estimator value with propagated error bars. |

In 1D the combination is `c_N = 2 M_n(N/2) ‚àí M_n(N)` and `M_n(N) = Œ£_k 2^k c_{N/2^k} + M_n(small)`. In 2D, use the subsystem-difference combinations from the topological-entanglement-entropy family: `Œ≥ M_n = M_n(L√óL) ‚àí 4 M_n(L/2√óL) + 4 M_n(L/2√óL/2)` and similar. The combination cancels volume-law (1D) or volume + area-law (2D) terms so the residual is `O(1)` or `O(log N)` and the variance becomes polynomial in `N` even for `n > 1`.

### Stage 3 ‚Äî convergence + variance diagnostics

| Input | Output |
|---|---|
| All chain outputs | `(N_S ‚Üí N_S/2)` reblocked errors, `œÑ_int(N)` curve, `œÉ(N)` curve. |

Variance bounds are content; this card just declares the diagnostic. See "Variance bounds" below.

| Knob | Effect | Starting point |
|---|---|---|
| `N_S` | Total Markov-chain length. Statistical error `~ 1/‚àö(N_S/œÑ_int)`. | `10‚Åµ`‚Äì`10‚Å∑`; set per the variance bound below and reblock to verify. |

| `ŒÝ_P` (target distribution) | `Œû_P` for `M_n` direct; `ŒÝ_{P,n}` for the increment construction. | Use `ŒÝ_{P,n}` whenever `n ‚âÝ 1` and the target is `M_n` directly; switch to the increment construction when `M_n` is volume-law. |

## Variance bounds (content per `knowledge-base/magic-conventions.md`)

- For `M_1`: `Var(M_1)` is at most quadratic in `N`; the estimator is *efficient* (polynomial cost). For `m_1 = M_1/N`, the standard error scales as `1/‚àöN`.
- For `M_n`, `n ‚âÝ 1`: when the SRE is volume-law (`M_n ~ N`), the variance grows exponentially with `N`. Use the increment / subleading construction (Stage 2) to cancel the volume-law term and recover polynomial cost. When the SRE is at most `O(log N)` (frustrated regimes, certain Bell-magic decompositions), the direct estimator is already efficient.
- For `L(œÅ_AB)`: efficient because `L` is bounded by mutual-information-like content; volume-law cancellation is automatic.

## Pitfalls

- **Critical slowing-down at criticality** ‚Äî `œÑ_int` grows near continuous QPTs. Diagnose via `œÉ(N)` curves; mitigate with cluster updates when implemented, or with longer chains and reblocked errors.
- **Sign of `L(œÅ_AB)`** ‚Äî SRE does not satisfy subadditivity, so `L(œÅ_AB)` is not sign-definite. Negative extrema at criticality are valid; do not flip signs to "fix" them.
- **Wrong symmetry sector in proposals** ‚Äî single-site proposals on a Z_n-symmetric state rapidly leave the symmetric subgroup and bias the estimator. Always use two-site proposals when symmetry-preserving Pauli strings require it.
- **Wrong proposal kernel identity** ‚Äî a symmetry-restricted Markov chain is defined by its generator/proposal kernel, not just its target distribution. Record the proposal identity in every manifest and make aggregators reject mixed or missing proposal metadata.
- **Heavy-tailed ratio observables** ‚Äî ratio estimators can have rare-event variance that simple fixed-block errors miss. Before scaling to MPS/TTN sizes, run an exact small-L support and ratio-variance diagnostic and treat block errors as provisional until reblocking is stable.
- **`ŒÝ_{P,n}` with small support** ‚Äî at large `N` and `n > 1`, `ŒÝ_{P,n}` concentrates on a tiny fraction of `P_N` and the chain may get stuck. The increment construction is the standard fix; cluster updates help further.
- **Finite-`N_M` bias** (experimental variant) ‚Äî finite shots introduce `O(1/N_M)` bias that *does not* shrink with `N_S`. Both `N_M` and `N_S` must be balanced; treat them as two independent convergence parameters via `/verify-convergence`.

## Deterministic Pauli-basis lift (variant)

For MPS wavefunctions, an alternative to the Markov chain is to represent `Tr(œÅP)` directly as an MPS over the Pauli index axis (`d^2`-dim per site) and contract with standard MPS routines. The resulting object ‚Äî the *Pauli-basis MPS* ‚Äî gives:

- **SRE** by treating `M_n` as a 2D tensor network and approximating its contraction with MPS algorithms.
- **Bell magic, stabilizer nullity** ‚Äî additional monotones that the Markov-chain protocol does not directly access.
- **Stabilizer-group identification** ‚Äî by reading off zero entries of the Pauli MPS.

The lift trades Markov-chain variance for tensor-network truncation error; convergence is governed by an additional bond dimension on the Pauli axis. It is the recommended runtime when (i) the MPS is already in hand, (ii) the target observable is a Pauli-MPS-friendly monotone (Bell magic, nullity), or (iii) volume-law magic with `n > 1` makes the Markov chain exponentially expensive even after the increment construction.

## Experimental-protocol variant (stages)

Per AGENTS.md "multi-stage orchestration lives in method cards", the experimental-protocol variant is declared as stages alongside the numerical variants. The chain machinery is the same; the wavefunction stage is replaced by hardware state preparation, and `Tr(œÅP)` is replaced by a finite-shot Pauli-measurement estimate `PÃÑ`.

### Stage E0 ‚Äî Hardware state preparation

| Input | Output |
|---|---|
| Hamiltonian, target sector, circuit specification (variational ansatz, adiabatic schedule, or analog evolution), hardware backend | A reproducible state-preparation circuit + calibration data (qubit fidelities, gate errors, readout-correction matrix). |

Driver: hardware-control stack (out of scope for this card). The harness consumes the prepared state as a black box that exposes Pauli-measurement capability. Document the preparation protocol in the run report ‚Äî fidelity claims propagate to the SRE error budget downstream.

### Stage E1 ‚Äî Pauli-measurement scheduling

| Input | Output |
|---|---|
| Stage E0 prepared state, current Pauli string `P` from the chain, shot budget `N_M` per Pauli | Measurement record `{m_k}_{k=1..N_M}` of `¬±1` outcomes for `P`; finite-shot estimate `PÃÑ = (1/N_M) Œ£_k m_k`. |

Driver: hardware-control stack with single-Pauli measurement primitive. For `d > 2` qudits, replace `¬±1` with the appropriate clock-eigenvalue distribution; the estimator structure is unchanged. Apply readout-error correction from the calibration matrix at this stage.

The chain proposes `P ‚Üí P'`; Stage E1 measures `P'` only when `PÃÑ'` is needed (most chain steps reuse prior measurements via cached `PÃÑ` lookups). Effective shots per chain step is therefore lower than `N_M` √ó chain length; cache size is a runtime knob.

### Stage E2 ‚Äî Markov-chain estimator on `PÃÑ`

| Input | Output |
|---|---|
| Stage E1 finite-shot estimates `PÃÑ` over `N_S` Pauli strings, target distribution `ŒÝ = Œû_P` (or `ŒÝ_{P,n}`) | Chain output `results/<run>/N=<N>/chain.h5` with samples, log-weights, autocorrelation diagnostics, finite-shot bias / variance decomposition. |

Same Algorithm-1 Metropolis as Stage 1 of the numerical variant, but the target distribution evaluator now consumes `PÃÑ` instead of exact `Tr(œÅP)`. Acceptance probability uses `|PÃÑ|^{2n}` directly; finite-`N_M` bias propagates through to the estimator.

### Stage E3 ‚Äî Bias / variance budget reconciliation

| Input | Output |
|---|---|
| Stage E2 chain output, `(N_M, N_S)` knobs, target accuracy on `m_n` | Decomposed error budget: `Œ¥m_n^{shot} ~ 1/‚àöN_M` (per-Pauli) + `Œ¥m_n^{chain} ~ 1/‚àö(N_S/œÑ_int)` (chain) + `Œ¥m_n^{bias} ~ O(1/N_M)` (does *not* shrink with `N_S`). Recommended `(N_M, N_S)` adjustment if budget exceeded. |

The finite-`N_M` *bias* (not variance) is the distinct feature of this variant: increasing `N_S` alone does not reduce the bias. Both `N_M` and `N_S` must be balanced; the standard diagnostic is a 2D scan over `(N_M, N_S)` via `/parameter-scan` paired with `/verify-convergence`.

### Variant verification

- **Limit check (zero shots)**: at `N_M ‚Üí ‚àû`, the variant must reduce to the exact-`Tr(œÅP)` chain estimator; cross-check on a small system.
- **Limit check (shot-noise dominated)**: at `N_M ~ 1`, the bias dominates; the SRE estimate is biased low by an `O(1/N_M)` term independent of `N_S`. Verify the bias scaling explicitly via the `(N_M, N_S)` 2D scan.
- **Cross-method check**: re-run the same prepared state via the numerical variant (Stage 0‚Äì3) at a small `N` where the wavefunction is constructible; the experimental and numerical estimators must agree once shot bias is accounted for.
- **Benchmark anchor**: see `knowledge-base/magic-benchmarks.md` for the reference `(N_M, N_S)` regime where the experimental protocol has been demonstrated to reproduce the numerical SRE within budget.

### When to use this variant

- The user is in an experimental shot-budget question (Fig-10/11-class diagnostic).
- The user is benchmarking a hardware platform's stabilizer-detection capability.
- The user has measurement records on hand (no wavefunction available); this variant is the only path.

## Verification (per-method, complements skill-level verification)

- **Single-qudit limit** ‚Äî for product states of single qudits, the chain estimator must reproduce the analytic value (see `knowledge-base/magic-conventions.md` qudit examples).
- **Stabilizer-state limit** ‚Äî `M_n = 0` for any stabilizer state; chain estimator should return 0 within statistical error.
- **`œá`-convergence** ‚Äî sweep the wavefunction's bond dimension and confirm the SRE estimate asymptotes; this is `/verify-convergence` over `œá` and is the diagnostic for FIG. 13-class checks.
- **`N_S`-convergence** ‚Äî reblocked error decreases as `1/‚àö(N_S)` once the chain is past `~10 √ó œÑ_int`.
- **Cross-method check** ‚Äî TTN-based and MPS-based chains must agree on the same Hamiltonian within both methods' truncation budgets. The deterministic Pauli-basis lift is a third independent path.
